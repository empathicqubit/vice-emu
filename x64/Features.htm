Features of the VICE Commodore 64 Emulator (x64)

The C64 emulator, called 'x64', features a fairly complete emulation of the VIC-II video chip: sprites, all registers and all video modes are fully emulated. Custom palettes, gamma, contrast, and several other settings are configurable.The emulation has been fully cycle-accurate since version 0.13.0.

A rather complete emulation of the SID sound chip is also provided. All the basic features are implemented as well as most of the complex ones including synchronization, ring modulation and filters. There are two emulators of the SID chip available: one is the "standard" VICE emulator, available since VICE 0.12; the other one is Dag Lem's reSID engine. The reSID engine is a lot more accurate than the standard engine, but it is also a lot slower, and only suitable for faster machines. Dual SID chips are also emulated and can be enabled in a configurable address space.

Naturally, both CIAs (or VIAs, in some cases) are fully emulated and cycle accurate.

Kernal, basic, character, and drive roms can be swapped out in favor of 3rd party replacement roms. Some rom sets known to work are JiffyDos, DolphinDos, SpeedDos, and others.

NTSC and PAL modes are also fully emulated.

A very powerful and robust ML monitor and debugger is built in that includes support not only for the main machine but also for emulated disk drives.

x64 also supports a large number of additional hardware devices and addons. Here's a good rundown of most of them:

  • Full 4 drive IEC emulation of the 1541, 1541-II, 1570, 1571, and 1581. For drives that support such addons, emulation also includes up to 40k drive ram expansion, 35 -> 40 track expansion, and userport parallel support for various hardware drive speeders. In additon, virtual device numbers can be mapped to a directory on the host PC to provide access to the host's filesystem. Real physical IEC drives can also be used in the emulator with the correct interface and OpenCBM drivers installed.
  • IEEE488 emulation which allows additional drive types to be emulated. (2031, 2040, 3040, 4040, 1001, 8050 and 8250)
  • CBM Datasette emulation is included and has options for tweaking the virtual motor speed.
  • Mass storage options including the IDE64 and MMC64
  • Userport and IEC printer emulation. Ascii, MPS803, and NL10 virtual printer emulation can print to printers attached to the PC, a text file, and a real IEC printer via OpenCBM
  • Fully customized keyboard mappings, dual joystick mappings, and 1351/Neos/Amiga mouse input on either JoyPort
  • Configurable C64 borders and sprite collision detection
  • Userport RS-232 emulation and support for ACIA RS-232 cartridges like the Swiftlink and Turbo232 with communication over tcpip
  • Ethernet support using RR-Net and TFE cart emulation
  • Up to 16 megabyte REU and 4 megabyte GEORam support with saving of the image to the host filesystem
  • Several other ram additions and hacks are emulated including the Ramcart, Plus60k, Plus256k, and 256k
  • Digimax support at a configurable address
  • Generic support for CRT images, plain binary rom images and many different cartridge types and configurations can be used such as Action Replay, Retro Replay, Expert Cartridge, Epyx Fastload, The Final Cartridge, and Super Snapshot

New features and support for additional hardware is being added all the time. If you have a new feature request, please dont hesitate to visit the Sourceforge page and add it to the tracker.
